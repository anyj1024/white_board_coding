# 三种单例模式

## 1. Eager Initialization (饿汉式初始化)

- 特点：
    - 在类加载时就创建实例
    - 天然线程安全
    - 简单直接

- 优点：
    - 实现简单
    - 没有线程同步问题

- 缺点：
    - 可能造成资源浪费
    - 无法处理异常

## 2. Lazy Initialization (懒汉式初始化)

- 特点：
    - 第一次调用时才创建实例
    - 使用同步方法确保线程安全

- 优点：
    - 节省资源
    - 需要时才创建

- 缺点：
    - 每次获取实例都要同步，可能影响性能

## 3. Double-Checked Locking (双重检查锁定)

- 特点：
    - 结合了懒加载和性能优化
    - 使用 volatile 关键字

- 优点：
    - 延迟加载
    - 减少同步开销，提高性能

- 缺点：
    - 实现较复杂
    - 在某些 Java 版本中存在隐患
